# Creative Node Editor (CNE) 软件说明文档

## 1. 软件概述
Creative Node Editor (CNE) 是一款面向创意编程和视觉逻辑设计的节点式编辑器。它允许用户通过图形化的节点连接，构建、渲染并管理复杂的视觉场景与数据流。本软件旨在提供一个轻量级、高性能且易于扩展的创意生产工具。

## 2. 软件架构
CNE 采用了高度模块化的软件架构，核心组件包括：

- **Core (核心逻辑层)**: 
    - `NodeCanvas`: 负责画布的交互设计，包括缩放、平移、选择、拖拽等。
    - `ConnectionManager`: 管理节点间的拓扑结构，负责端口连接与拓扑排序。
    - `RenderEngine`: V2 版本的渲染流水线，将 `Renderable` -> `RenderPool` -> `Renderer` 串联，支持实时评估与输出。
    - `Clipboard`: 高级剪贴板管理，支持带连接关系的批量节点复制与粘贴。
- **Nodes (节点层)**: 
    - 采用继承体系，所有节点均派生自 `Node` 基类。
    - 包括：`ImageNode` (渲染)、`TextNode` (数据)、`TransformNode` (变换)、`ContainerNode` (逻辑容器) 等。
- **UI (用户界面层)**: 
    - 纯原生 CSS/JS 构建，遵循现代深色系设计语言。
    - `ParamPanel`: 动态属性面板，根据节点类型自适应生成控件。
    - `Minimap`: 全局视口导航。

## 3. 软件功能与特点
- **高效的拓扑求值**: 采用自动拓扑排序，确保逻辑环路的正确处理与实时反馈。
- **动态变换系统**: 用户可以输入 JS 表达式（如 `$x * Math.sin(t)`) 直接驱动物体变换。
- **多对象编辑**: 支持多选、批量移动、批量删除，以及**带拓扑结构的批量复制粘贴**。
- **全屏创意演示**: 支持视口全屏与预览面板全屏（F11 兼容），适用于现场演示。
- **工程持久化**: 自定义 `.cne` 格式，支持序列化/反序列化。
- **本地化隐私优先**: 采用 Local-Only 资产管理，避免网络延迟与跨域问题带来的不确定性。

## 4. 软件使用细则
1. **添加节点**: 在右键菜单或编辑菜单中选择“新建节点”。
2. **连接节点**: 从节点的输出端口（右侧）拖拽线条至输入端口（左侧）。注意：系统会自动检查数据类型兼容性。
3. **调整参数**: 选中节点后，在右侧属性面板调整。点击属性左侧的小圆圈可开启/关闭该属性的外部输入。
4. **演示模式**: 点击预览面板右上角的“⛶”进入全屏渲染模式。

## 5. 开发过程记录
- **V1.x**: 基础架构搭建，实现单点连接与 Canvas 渲染。
- **V4.x - V6.x**: 引入 V2 渲染流水线，重构属性面板，增加几何变换节点。
- **V7.x**: 强化字符串处理与本地文件支持，引入快捷键系统。
- **V8.0++**: 完善多对象编辑逻辑，实现复杂连线结构的深度复制，并优化了 `ImageNode` 的内存加载稳定性。

## 6. 研究意义
本研究旨在探索 Web 端高性能节点式编辑器的实现范式。通过纯 Vanilla JS 构建复杂系统，证明了 Web 原生 API 在处理创意编程与实时渲染任务中的巨大潜力，为创意工作者提供了一套轻量级、零依赖的逻辑原型构建工具。

## 7. 有待处理的问题与优化计划
### 有待处理的问题
- **本地图片持久化局限**: [IMPORTANT] 受限于浏览器安全机制，工程加载时无法自动提取并加载本地绝对路径的图片。**目前在加载工程后，用户需要通过点击图片节点的文件夹图标重新手动关联一次图片。**
- **性能瓶颈**: 在处理数千个微型粒子节点时，Canvas 2D 的 CPU 绘图开销依然较大。

### 优化计划
- **WebGL/GPU 渲染**: 计划将底层渲染引擎升级至 WebGL2 或 WebGPU，以支持大规模粒子流。
- **撤销/重做 (History)**: 引入强一致性的历史记录管理栈。
- **节点分组 (Groups)**: 支持将多个节点打包成一个 Sub-Graph，实现逻辑复用。
- **路径记忆自动重载**: 研究通过本地中转 Server 或 File System Access API 解决图片自动重载问题的可行性。
